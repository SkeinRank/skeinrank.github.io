title: SkeinRank current status
description: What is available in the SkeinRank core, what remains a platform beta, and how to evaluate it honestly.
---

SkeinRank is an actively developed open-source project. The local SDK and CLI are the lowest-friction adoption path; the governance platform remains a beta environment for evaluation and pilot work.

## Available now

- Python SDK and CLI for dictionary validation and canonicalization
- local document helpers and dictionary onboarding workflows
- terminology discovery and drift reports with context-aware evidence
- explainable passport and debug output
- profiles, aliases, slots, bindings, and immutable snapshot models
- reproducible Apache Airflow documentation benchmark artifacts

## Platform beta

- PostgreSQL-backed governance API
- React governance console
- evidence-backed proposal review
- Elasticsearch bindings, dry-runs, enrichment jobs, and alias-swap metadata
- snapshot audit and rollback metadata
- Search Playground for binding-aware query preview
- integrations graph for Profile → Binding → Index → Snapshot topology
- Docker Compose quickstart for local evaluation

## Not claimed yet

- a hosted multi-tenant SaaS product
- a complete production hardening and upgrade guide
- universal benchmark accuracy across all corpora
- automatic synchronization with every repository lexicon
- a replacement for semantic retrieval or the application search engine

## Recommended evaluation path

1. Run the local SDK quickstart.
2. Test dictionary onboarding and drift discovery on a corpus you understand.
3. Review evidence and known false-positive classes.
4. Start the Docker beta only when runtime bindings and snapshot governance are relevant.
5. Treat any production rollout as an explicit pilot with rollback and observability.
