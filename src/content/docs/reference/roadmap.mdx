This roadmap keeps the public story clear without overselling unfinished pieces.

## Near term

- Keep the local SDK/CLI path stable.
- Add concise examples for dictionary validation, extraction, and canonicalization.
- Add screenshots of the governance UI after the end-to-end case is verified.
- Add a minimal before/after retrieval demo.
- Add generated Python API reference after public interfaces settle.

## Platform validation

- Run PostgreSQL-backed governance API locally.
- Run UI against the API with auth enabled.
- Validate dictionary import/export.
- Test suggestion approval with evidence snapshots.
- Run Elasticsearch dry-runs and enrichment jobs.
- Validate alias swap and rollback flow.

## Later

- Production deployment notes.
- More provider examples.
- Versioned docs.
- Public demo dataset.
- Benchmarks or evaluation pages.
- Governance playbook for teams.
